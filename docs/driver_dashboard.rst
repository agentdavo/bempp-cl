Driver Dashboard (How To Read)
==============================

The interactive driver/waveguide dashboard is generated by
`bempp_audio.viz.save_driver_dashboard_html(...)` and written as a standalone
HTML file.

If a dashboard looks “empty”, verify that the HTML is opening with access to the
embedded Plotly JS (default) and that the `FrequencyResponse` contains frequency
points.

Data Flow (High Level)
----------------------

1. `Loudspeaker.build()` produces a `SimulationRequest`.
2. `bempp_audio.api.solve.solve_request(...)` runs the BEM solve and returns a
   `bempp_audio.results.FrequencyResponse` containing `RadiationResult` objects.
3. `save_driver_dashboard_html(...)` computes:

   - SPL curves and polar map via `FrequencyResponse.polar_sweep(...)`
   - DI and beamwidth vs frequency via `FrequencyResponse.directivity_index_vs_freq(...)`
     and `FrequencyResponse.beamwidth_vs_freq(...)`
   - impedance traces (best-effort) from each `RadiationResult`

Panels
------

Scorecard
  Quick summary of sweep range and graded metrics.

SPL Curves
  SPL vs frequency at selected angles plus a listening-window (energy-average)
  curve.

Polar Map (normalized)
  SPL heatmap over angle vs frequency, normalized to the selected
  `normalize_angle_deg`.

Directivity Index / Beamwidth
  DI and -6 dB beamwidth vs frequency.

Impedances
  - Driver electrical impedance (when driver metrics are present)
  - Throat acoustic impedance and specific radiation impedance (computed from the BEM results when available)

Throat Impedance Normalization
  When waveguide metadata is available, the dashboard normalizes throat acoustic impedance by the
  plane-wave characteristic impedance at the throat:

  ``Z0 = ρ c / S_throat`` (units Pa·s/m³)

  and plots ``Z_throat / Z0``:

  - One y-axis: ``Re(Z/Z0)``, ``Im(Z/Z0)``, and ``|Z/Z0|`` with a default range of **0–2**

Radiation Impedance Normalization
  The “Radiation Impedance” panel in the matplotlib reports (and the “Radiation Zspec” panel in the dashboard)
  uses the *specific* radiation impedance ``Zspec`` (units Pa·s/m). When medium properties are available, it is
  normalized by ``ρc`` so the plotted quantity is dimensionless:

  ``Zspec / (ρc)``

  The default y-axis range is **0–2** in the normalized mode.

Radiated Power / Group Delay
  - Radiated power vs frequency (when available)
  - On-axis group delay vs frequency (computed from the complex on-axis field)

Recommended Targets (Practical Defaults)
----------------------------------------

The defaults in `bempp_audio.viz.plotly_dashboard.DashboardOptions` are intended
as conservative “sanity” thresholds for smoothness in typical horn/guide work:

- Listening-window SPL ripple (peak-to-peak in bands): PASS ≤ 2 dB, WARN ≤ 3 dB
  across 1–2 / 2–5 / 5–10 / 10–20 kHz.
- DI ripple (peak-to-peak in bands): PASS ≤ 2 dB, WARN ≤ 3 dB.
- Beamwidth monotonicity: PASS ≥ 0.70, WARN ≥ 0.60 (higher is smoother/more monotone).
- Peak excursion: PASS ≤ 0.20 mm, WARN ≤ 0.30 mm (design-stage guardrail; tune per driver).

These are not “standards”; adjust them to your use case, driver size, and
measurement conventions.

Notes / Common Pitfalls
-----------------------

- Single-frequency sweeps will still render, but many “ripple” style metrics are
  not meaningful with <2 frequency points.
- If Plotly JS is loaded from a CDN and you open the HTML without network
  access, the page will appear blank. Prefer embedded Plotly JS (default).
